📋 DEPLOYMENT CHECKLIST (7 STEPS)
═══════════════════════════════════════════════════════════════════════════

STEP 1: CREATE GITHUB REPO
   [ ] Go to https://github.com/new
   [ ] Name: RAG_PIPELINE
   [ ] Choose: Public
   [ ] Click "Create repository"
   [ ] Run: git remote add origin https://github.com/YOUR_USERNAME/RAG_PIPELINE.git
   [ ] Run: git branch -M main
   [ ] Run: git push -u origin main

STEP 2: GET API KEY (Pick ONE)
   [ ] Google Gemini: https://makersuite.google.com/app/apikey (Free!)
   [ ] OR OpenRouter: https://openrouter.ai/keys (Cheapest)
   [ ] OR OpenAI: https://platform.openai.com/api-keys (Best quality)

STEP 3: CREATE RENDER ACCOUNT
   [ ] Go to https://dashboard.render.com
   [ ] Sign up (easier with GitHub)

STEP 4: DEPLOY TO RENDER
   [ ] Click "+ New" → "Web Service"
   [ ] Choose your GitHub repo
   [ ] Fill in form (name: smart-rag-app, env: Docker, region: Oregon)
   [ ] Click "Create Web Service"
   [ ] Wait for build to complete (3-5 min)

STEP 5: ADD API KEY TO RENDER
   [ ] Click "Environment" tab
   [ ] Add LLM_PROVIDER, LLM_MODEL, and API key
   [ ] Click "Save" for each variable
   [ ] Render restarts automatically

STEP 6: VERIFY IT WORKS
   [ ] Open: https://smart-rag-app.onrender.com/
   [ ] Upload a test document
   [ ] Ask a question
   [ ] Verify streaming response works

STEP 7: SHARE YOUR APP!
   [ ] Get your personal URL from Render
   [ ] Share: https://smart-rag-app.onrender.com/
   [ ] Anyone can use it (no installation needed!)

❓ FREQUENT QUESTIONS
═══════════════════════════════════════════════════════════════════════════

Q: Will my data be saved?
A: SQLite persists during session, but resets on redeploy.
   Use PostgreSQL (Render add-on) for permanent storage.

Q: Can multiple people use it?
A: Yes! Share the URL. Each person sees the same documents.

Q: How do I update the app?
A: Push to GitHub. Render auto-deploys within 1-2 minutes.

Q: What if it breaks?
A: Check Logs in Render. Restart from dashboard.

Q: Can I use my own domain?
A: Yes. Render Settings → Custom Domain.

Q: How do I add more features?
A: Edit code locally, push to GitHub, Render auto-deploys.

Q: Is it secure?
A: API keys stored in Render (encrypted). No data saved locally.

Q: Can I use different LLM providers?
A: Yes! Configure in environment variables. Switch anytime.
